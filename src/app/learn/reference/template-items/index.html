Template items

This reference describes the default items that the template includes in a SKY UX single-page applicatoin after you create a SKY UX project.

After you create a SKY UX project, the skyuxconfig.json file in your project's directory allows you to configure settings for SKY UX. All properties in skyuxconfig.json are optional, and by default, the file only specifies values for the mode and compileMode settings.

The skyuxconfig.json does not include all SKY UX configuration options. SKY UX Builder defines some configuration options that are not included in this file.

The default items that the template creates in a SKY UX SPA include:

  • vscode folder — Contains 2 .json files to configure settings that are included for the developer's convenience.

    • launch.json file — A launch configuration file that provides settings for how to debug the build process for the SPA.

    • settings.json file — A file that controls editor settings for VSCode.

  • coverage folder — Stores code coverage results for unit tests. This folder is generated when you run skyux test or skyux watch and provides a convenient way for developers to see what code their tests did and did not reach. We recommend that all new code should have 100 percent unit test code coverage. For information about the code coverage requirements to contribute to SKY UX, see the SKY UX 2 contribution guidelines.

  • dist folder — Contains files that are deployed through the continuous integration pipeline. This folder is generated when you run skyux build. The CI layer of SKY UX runs unit tests, visual regression tests, and end-to-end tests and handles staging for additional testing and validation.

  • e2e folder — The required location to store end-to-end tests. As a convenience, this folder contains sample tests.

  • node_modules folder — Stores the external dependencies for SKY UX components and SKY UX Builder. This includes system files to build the single-page application. The folder is listed in the .gitignore file. Do not place it into source code control.

  • src folder — Contains the source files for the SPA.

  • app folder — XXXXXX.

    • about folder — Contains the page, component, and unit test for the About page in the template SPA. The index.html file is required to generate a route, while the other .html file and the .ts files create the AboutComponent that index.html references. The .spec.ts file is the unit test for the component, and we recommend storing the unit test in the same location as the component .ts file.

    • shared folder — The recommended location for components and services that are shared across an application. In the template project, we include an app-nav component that creates the navigation menu.

    • app-extras.module.ts — Handles tasks such as registering providers and entry components. If this file exists, SKY UX Builder automatically imports it into the build process.

    • home.component.html, home.component.ts, home.component.spec.ts, and index.html — The page, component, and unit test for the Home page in the template SPA. The index.html file is required to generate a route, while the other .html file and the .ts files create the HomeComponent that index.html references. The .spec.ts file is the unit test for the component, and we recommend storing the unit test in the same location as the component .ts file.

  • .editorconfig — Configurations settings for editors that support an EditorConfig file to define and maintain consistent coding styles between editors and IDEs. For information about EditorConfig files, see the EditorConfig website.

  • .gitignore — A standard file for Git repositories that lists untracked files to intentionally ignore in the working directory. For information about gitignore files, see the Git documentation.

  • package.json — A configuration file for NPM to specify dependencies that are stored in the node_modules folder. For information about package.json files, see the NPM documentation.

  • README.md — A Markdown file for the repo to describe the SKY UX application.

  • skyuxconfig.json — A configuration file to configure settings for SKY UX applications. For information about these configuration options, see the configuration options reference.

  • tsconfig.json — A configuration file for the TypeScript compiler. For information about tsconfig.json files, see the TypeScript documentation.

  • tslint.json — A configuration file for TSLint. For information about tslint.json files, see the TSLint documentation.
