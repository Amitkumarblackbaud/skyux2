The SKY UX template provides the skyuxconfig.json file to specify configuration settings, but you can also create additional config files to apply configuration settings specifically to SKY UX CLI commands. For example, you can create skyuxconfig.serve.json to apply specific configuration settings when skyux serve runs.

For example, to apply configuration settings when skyux serve runs, create a skyuxconfig.serve.json file and specify the configuration settings. When skyux serve runs, it automatically includes and merges in skyuxconfig.serve.json if it exists. Create configuration file for a SKY UX CLI command

  1. In a local editor, create a configuration file in the root directory alongside the project's default skyuxconfig.json file. Follow the skyuxconfig.[COMMAND].json naming convention.

    For example, to apply configuration settings when skyux e2e runs, create skyuxconfig.e2e.json.

  2. In the new config file, specify the configuration settings

    For example, you can specify a mock server URL to use when you run skyux e2e.

    "appSettings": { "mockServerURL": "https://localhost:9000" }

  3. Save your changes. SKY UX automatically reads files with the skyuxconfig.[COMMAND].json naming convention. For example, when skyux e2e runs, SKY UX automatically includes and merge in the skyuxconfig.e2e.json.
