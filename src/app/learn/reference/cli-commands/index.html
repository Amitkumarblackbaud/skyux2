This reference describes the commands that are available in the SKY UX CLI.

  • skyux new — Initializes a new local SKY UX project. For more details, see the skyux new topic.

  • skyux serve — Serves the current SKY UX single-page application locally. For more details, see the skyux serve topic.

  • skyux build — Builds the current SPA into the dist/ folder. For more details, see the skyux build topic.

  • skyux test — Runs unit tests. For more details, see the skyux test topic.

  • skyux e2e — Runs end-to-end tests. For more details, see the skyux e2e topic.

  • skyux watch — Runs the test command and watches the file system for changes. For more details, see the skyux watch topic.

  • skyux lint — Runs the TSLint extension against the files in the src folder. For more details, see the skyux lint topic.

  • skyux build-public-library — Bundles custom Angular components into a consumable NPM module. For more details, see the skyux build-public-library topic.

  • skyux version or skyux -v — Returns the version of SKY UX CLI.

  • skyux, skyux help, or skyux -h — Displays help information for SKY UX CLI arguments. This lists the commands and options that are available in the SKY UX CLI and provides descriptions.
